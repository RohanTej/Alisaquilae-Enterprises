Alisaquilae Enterprises - Website v1.0 - Manual


This documentation will help you go over the website and do the edits from the backend. This will go over the important changes you will have to make in order to get the website up and running in no time.


Languages used :

  1. HTML5
  2. CSS3
  3. PHP
  4. AJAX
  5. JavaScript - jQuery library

Additional features used :

  1. Bootstrap (Free)

Folders

  • assets
  • css
  • fonts
  • js
  • main directory

assets

This folder contains the essintial website files like pictures, logos etc. This file currently contails only the logo and loader.

css

This folder contains files require to style the entire website. The files containing the name bootstrap are all bootstrap related css files.
Whislt style.css is makes modifications to bootstrap or adds custom styling to the website.

fonts

This folder contains the fonts and icons(glyphicon) used through out the website. You can add and link additional fonts in this folder.

js

This folder contains JavaScript from BootStrap and CanvasJS. It also has home.js which has the code to execute the page home.php properly. md5.js is used to convert string into encrypted md5 hash. It is only used to encrypt passwords.

main directory

  • .htaccess can hide the .php extension from the url which improves the security and SEO.
  • connect.php has the credentials to connect to the SQL server to retrive user's information on demand.
  • error.php is loaded when there is an error submitting the registration form where as success.php is loaded when there was no error and the form is successfully submitted.
  • favicon.ico is recognised by the browser as the default favicon to display on the tab. If you wish to change it then replace the file with your favicon.ico and make sure it is 60x60
  • functions.php contains the various functions written in php which can be called from anywhere on the website if you use the line Include 'functions.php'; in your php code.
  • home.php is the main dashboard of the website. From here, the user can call in diffrent pages under one master page without loading a new window. This grants a seemless exprience for the user.
    • loading.php contains the ajax-loader.gif located in assets. This page is loaded in home.php whilst the other page user requested to indicate that the page is loading.
    • homei.php, orderi.php, reporti.php, supporti.php or coupi.php will be loaded with the user requestes for the respective page
  • index.php is the front page of the website which contains the login box. If the user is already logged in then he will be redirected to home.php
  • login.php and signup.php is the frontend which the user will see to login or signup respectively.
  • login_a.php and sign_a.php is the backend which the user will not see, but will be loaded with ajax to send request to SQL server to retrive or submit data to login or signup respectively.
  • template.php is a template code which can be used to create a new page with ease.
